Проблема

ЛСУМАДИ работает с клиентами, у которых сразу несколько сотрудников вовлечены в ремонт и согласования. Они привыкли вести всё в групповом WhatsApp чате, а в RetailCRM доступен только стандартный один‑к‑одному канал. Результат — разрывы контекста, время на пересылку и потерянные детали.

  • Нет групп в стандартной интеграции.
  • Ручное создание чатов замедляет ответ.
  • Нельзя «написать первым» без шаблонного окна.
  • Повторяющиеся вводные для каждого контакта.

Что было критично

  • Авто-чаты при любом создании клиента.
  • Одна лента для нескольких участников.
  • Понятно кто пишет.
  • Соблюдение правил WhatsApp.
  • Ноль лишних действий менеджера.

Решение в двух словах

Лёгкий proxy на Node.js между 360dialog и RetailCRM: автостарт чата шаблоном и «виртуализация» группы — несколько людей → один чат с пометками имени.

1. Автостарт

При любом создании клиента отправляется визиточный шаблон — окно 24ч открыто.

2. Виртуальная группа

Участники пишут — менеджер видит одну чистую ленту.

3. Минимум трения

Без новых кнопок в CRM. Всё в стандартной карточке.

4. Контроль окна

Proxy отслеживает таймер и подсказывает когда нужен шаблон.

Высокоуровневая архитектура

Простая и расширяемая. Фокус — прозрачность и скорость.

Общий контур

Контроль 24‑часового окна

  • 01Шаблон отправлен
    Старт отсчёта.
  • 02Ответ клиента
    Окно обновлено.
  • 03Истёк таймер
    Сигнал менеджеру.
  • 04Новый шаблон
    Цикл заново.

Почему так

  • Минимум кода — проще сопровождать.
  • Нет тяжёлых зависимостей → скорость.
  • Готово к расширению (аналитика, боты).
  • Прозрачный поток без «магии».
