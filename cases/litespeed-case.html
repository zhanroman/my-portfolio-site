Старт: что было «до»

Сайт на WordPress + Elementor с тяжёлой темой и набором разношёрстных плагинов. Использовался обычный Apache (shared / без тонкой настройки). Скорость загрузки на мобильных «падала» — пользователи уходили, не дождавшись полного рендера.

  • PageSpeed Insights (Mobile) ~28–32.
  • LCP ~5.4s (медленный hero + крупный фон).
  • TTFB ~780ms (нагрузка + неэффективные плагины).
  • Requests ~180+, общий вес страницы ~6.3MB.
  • JS bundle ~3.4MB (дубли и неиспользуемые скрипты).
  • CLS 0.18 (всплывающие блоки меняли layout после загрузки).
Основные боли:
1) Перегруз от Elementor + плагинов.
2) Большие изображения и тяжелые видео в hero.
3) Никакого серверного кэша и HTTP/3.
4) Внешние скрипты блокировали рендер.
5) Нет системной стратегии по lazy / critical CSS.

Пошаговый план

  1. Диагностика

    Lighthouse, WebPageTest, Chrome DevTools (Coverage), Query Monitor, список всех подключений.

  2. Инфраструктура

    Миграция на VDS + OpenLiteSpeed (HTTP/3, LSCache), настройка Brotli, Object Cache.

  3. Чистка

    Удаление лишних плагинов, отключение неиспользуемых Elementor виджетов.

  4. CSS & JS

    Critical CSS inline, defer/async, объединение фрагментов, удаление jQuery‑Migrate.

  5. Медиа

    Перекодировка WebP/AVIF, responsive srcset, lazy, lite-video.

  6. Переразметка

    Фиксированные контейнеры (aspect-ratio) → снижение CLS.

  7. Финиш & Мониторинг

    Проверка повторно, настройка регулярных прогонов.

Сервер и инфраструктура

Что сделано

  • Переезд: Apache → OpenLiteSpeed (HTTP/3, events architecture).
  • Настройка LSCache (public + частичные исключения).
  • PHP оптимизация: OPcache, правильный memory_limit.
  • Brotli + Gzip fallback.
  • Object Cache (Redis) для повторных запросов.
  • aaPanel удобное управление + автоматические бэкапы.
Эффект: снижены TTFB и время генерации HTML; последующая фронтенд оптимизация стала ощутимее.

Фронтенд оптимизация

  • Выделен Critical CSS (верх экрана) → inline.
  • Остальной CSS подгружается позже (print media + onload switch).
  • Preload для ключевого шрифта (subset woff2).
  • Defer не критичных скриптов + async внешних виджетов.
  • Удалена блокирующая цепочка (emoji, блок‑стили WP).
Результат: First Contentful Paint быстрее, LCP сократился (из-за раннего CSS + оптимизации hero медиа).

Изображения и видео

  • Перекодировка в WebP / AVIF.
  • srcset + sizes для адаптивной загрузки.
  • Lazy loading + placeholders (blur / цветной блок).
  • Lite YouTube embed (загрузка iframe по клику).
  • Оптимизация hero фонового изображения (compress + правильные размеры).
Экономия: минус ~70% веса изображений и серьёзное сокращение трафика на мобильных.

Сокращение и оптимизация JS / CSS

  • Удалены лишние плагины (N → M) и виджеты Elementor.
  • Отключены wp-block-library / classic-theme-styles / jQuery Migrate.
  • Группировка мелких inline скриптов.
  • Перенос аналитики в конец + async.
  • Контроль за third-party: preconnect / dns-prefetch.
Итог: JS payload ↓, Total Blocking Time ↓, быстрее интерактив.

Кэш & Доставка

  • LSCache: страницы + объектный кеш.
  • Правильные заголовки Cache-Control для статических ресурсов.
  • Сжатие Brotli → Gzip fallback.
  • Установка preconnect к доменам шрифтов / статик CDN.
  • Прелоад критических ресурсов (fonts, hero image).
Result: повторные визиты почти мгновенные, TTFB упал.

Примеры кода и приёмов

Отключение ненужных стилей/скриптов WP 

add_action('wp_enqueue_scripts', function() {
wp_dequeue_style('wp-block-library');
wp_dequeue_style('wp-block-library-theme');
wp_dequeue_style('classic-theme-styles');
wp_dequeue_script('jquery-migrate');
}, 100);

Defer не критичных скриптов

function mark_defer($tag,$handle){
$defer = ['slider-js','analytics-js','forms-extra'];
if(in_array($handle,$defer)){
return str_replace(' src',' defer src',$tag);
}
return $tag;
}
add_filter('script_loader_tag','mark_defer',10,2);

Critical CSS + отложенная основная таблица 

<style>
/* critical (header, hero, nav) */
.header{display:flex;align-items:center;...}
.hero{min-height:60vh;display:grid;...}
.btn{display:inline-flex;align-items:center;...}
</style>
<link rel="preload" href="/assets/css/main.css" as="style">
<link rel="stylesheet" href="/assets/css/main.css" media="print" onload="this.media='all'">
<noscript><link rel="stylesheet" href="/assets/css/main.css"></noscript>

Риски и как они сняты

Риски

  • Поломка верстки после удаления плагинов.
  • Регресс CLS из-за ленивой загрузки.
  • Конфликт кеша с формами/динамикой.
  • Потеря аналитических событий при defer.

Решения

  • Staging проверка + сравнение скриншотов.
  • Фиксированные контейнеры (width/height, aspect-ratio).
  • Исключения LSCache (no cache для форм / auth).
  • Deferred инициализация analytics с fallback.

Результат и влияние

  • Скорость мобильной версии в зелёной зоне.
  • Холостой трафик (лишние запросы) сокращён.
  • Снижение вероятности отказов на первом экране.
  • Готов фундамент под SEO (структура быстрее индексируется).
  • Обслуживание проще: меньше плагинов → меньше конфликтов.
Дополнительно: быстрее релизы небольших визуальных правок — CSS архитектура стала чище.
